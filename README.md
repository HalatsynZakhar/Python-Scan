# Images XML Generator

Скрипт відстежує зміни зображень через `watchdog` і автоматично
перегенеровує XML. Супервізор контролює Python, кожні 5 хвилин перевіряє Git,
встановлює оновлення та за потреби перезапускає Python. BAT перезапускає
сам супервізор після його аварійного завершення.

## Розгортання на Windows Server

Відкрийте PowerShell від імені адміністратора.

### Повне автоматичне встановлення

```powershell
$Installer = "$env:TEMP\FULL_SERVER_INSTALL.ps1"
Invoke-WebRequest "https://raw.githubusercontent.com/HalatsynZakhar/Python-Scan/main/FULL_SERVER_INSTALL.ps1" -OutFile $Installer
powershell.exe -NoExit -NoProfile -ExecutionPolicy Bypass -File $Installer
```

`FULL_SERVER_INSTALL.ps1` автоматично:

- встановить Git і Python 3.13, якщо вони відсутні;
- клонує проєкт у `C:\Python-Scan`;
- під час повторного запуску примусово оновить проєкт із `origin/main`;
- перед оновленням зупинить старе завдання та процеси проєкту;
- відкриє `config.json` і дочекається закриття Блокнота;
- створить віртуальне середовище `.venv`;
- встановить бібліотеки з `requirements.txt`;
- перевірить створення XML;
- створить і запустить завдання `ImagesXML`;
- вимкне обмеження часу виконання завдання;
- перевірить, що Python справді продовжує працювати.

Під час встановлення відкриється `config.json`. Заповніть його:

```json
{
  "images_dir": "D:\\ShareFiles\\public\\foto",
  "output_dir": "D:\\ShareFiles\\public\\xml",
  "xml_filename": "images_export.xml",
  "log_filename": "images_export.log",
  "max_log_lines": 2000,
  "images_base_url": "https://img.example.com/foto",
  "task_user": "SYSTEM",
  "allowed_extensions": [".jpg", ".jpeg"]
}
```

`images_base_url` — публічний кореневий URL фотографій, посилання на який
записуються всередину XML.

Обов'язково замініть `images_base_url` на реальну адресу фотографій.
`output_dir` задає спільну папку збереження XML і журналу. Імена файлів
задаються окремо через `xml_filename` та `log_filename`.
Збережіть файл і закрийте Блокнот — встановлення продовжиться автоматично.

Для локальних шляхів інсталятор автоматично створює папки та надає
користувачеві завдання необхідні права:

- читання та спостереження для `images_dir`;
- створення, зміну та видалення для `output_dir`;
- зміну каталогу `C:\Python-Scan` для Git-оновлень, бібліотек і
  діагностичних файлів.

За замовчуванням локальне завдання працює від `SYSTEM`. Значення можна змінити
через `task_user`.

Для UNC-шляхів, наприклад `\\server\share`, інсталятор запросить Windows-логін
і пароль користувача з доступом до спільної папки. Пароль використовується лише
для створення завдання та не зберігається в `config.json` або журналах. Права
на віддаленій папці мають бути заздалегідь надані на файловому сервері —
локальний інсталятор не може сам змінити віддалені SMB-права.

Вікно встановлення не закривається автоматично. У разі помилки її текст
залишиться на екрані, а повний журнал буде збережено в:

```text
C:\Python-Scan-install.log
```

Якщо завдання Windows запустилося, але Python завершився, інсталятор явно
повідомить про це. Локальна діагностика, доступна навіть за недоступності
мережевої папки:

```text
C:\Python-Scan\logs\supervisor-local.log
C:\Python-Scan\logs\python-output.log
C:\Python-Scan\logs\python-error.log
```

Якщо на Windows Server немає `winget`, інсталятор автоматично завантажить Git
і Python з офіційних джерел та перевірить цифрові підписи файлів.

## Керування

Перегляд журналу:

```powershell
Get-Content "D:\ShareFiles\public\xml\images_export.log" -Tail 100 -Wait
```

Зупинка та видалення завдання:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File C:\Python-Scan\scripts\stop_tasks.ps1
```

Команда повністю зупиняє Python, BAT-супервізор і автооновлення, видаляє
завдання Планувальника та PID-файл. Інші Python-процеси не зачіпаються.
Після завершення вікно залишається відкритим до натискання Enter.

Оскільки завдання видаляється, для повного відновлення автоматичної роботи
повторно запустіть:

```powershell
powershell.exe -NoExit -NoProfile -ExecutionPolicy Bypass -File C:\Python-Scan\FULL_SERVER_INSTALL.ps1
```

Ручний запуск супервізора без Планувальника:

```powershell
C:\Python-Scan\scripts\auto_update.bat main 5
```

Зупинити ручний запуск можна сполученням `Ctrl+C` або через
`stop_tasks.ps1`.

Вручну виконувати `git pull` на сервері не потрібно: супервізор сам виконує
примусову синхронізацію з `origin/main`, оновлює бібліотеки та перезапускає
Python. Локальні зміни відстежуваних файлів на сервері при цьому видаляються.

## Як це працює

- Артикул береться з імені файлу до першого `_`: `X36B.jpg`, `X36B_1.jpg` і
  `X36B_2.jpg` належать до артикулу `X36B`. Файл без суфікса вважається
  головним, числові суфікси сортуються як числа.
- Python одразу створює XML, а потім очікує подій файлової системи без таймера.
- На початку XML записуються дата й час створення з часовим поясом, кількість
  артикулів і фотографій. Це дозволяє одразу перевірити актуальність файлу.
- Час XML і журналів отримується від кількох світових NTP-серверів, а не з
  годинника Windows. Використовується зона `Europe/Kyiv`: UTC+2 узимку та
  UTC+3 улітку.
- Якщо NTP недоступний, скрипт не зупиняється: записує попередження та
  продовжує роботу із системним часом.
- Після синхронізації інтервали рахуються монотонним годинником, тому
  неправильний системний час не впливає на Git-перевірки та heartbeat.
- Після зміни, додавання, видалення або переміщення зображення XML
  перегенеровується.
- Файли `.xml`, `.log` і тимчасові `.tmp` завжди ігноруються спостерігачем і
  не спричиняють повторне створення XML.
- Супервізор перевіряє Git кожні 5 хвилин.
- Кожні 5 хвилин Python і супервізор незалежно підтверджують роботу у спільному
  журналі.
- Запис до спільного журналу захищений міжпроцесним блокуванням Windows.
- Спільний журнал зберігається як UTF-8 з BOM, щоб кирилиця правильно
  відображалася у браузерах.
- За наявності оновлень Python зупиняється, виконується примусова синхронізація
  з `origin/main`, видаляються зайві невідстежувані файли, встановлюються
  залежності та Python запускається знову.
- `config.json`, `.venv` і журнали не видаляються під час оновлення.

Інсталятор задає `ExecutionTimeLimit = PT0S`, тому Планувальник не зупинить
завдання через 72 години. Налаштування зберігається після перезавантаження та
повторного встановлення.

## Основні файли

- `FULL_SERVER_INSTALL.ps1` — повне встановлення сервера з нуля; знаходиться
  в корені проєкту.
- `scripts\auto_update.bat` — запуск і відновлення супервізора.
- `scripts\supervisor.ps1` — контроль Python та оновлень Git.
- `scripts\stop_tasks.ps1` — повна зупинка процесів і видалення завдання.
